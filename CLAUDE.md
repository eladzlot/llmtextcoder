A generic R package for **LLM-assisted qualitative text coding**. The researcher
writes a rubric as a plain-text prompt template; the package injects participant
texts, calls OpenAI, and returns structured scores as a flat CSV. Rumination
coding is the first use case but the engine is fully domain-agnostic.

## Full function reference

### `R/params.R`
| Function | Signature | Role |
|----------|-----------|------|
| `run_params()` | `(model = "gpt-4o", temperature = 0)` | Validated constructor for model settings. Returns a `run_params` list. Pass to every API-calling function. |

### `R/prompt.R`
| Function | Signature | Role |
|----------|-----------|------|
| `read_template()` | `(path)` | Read a `.txt` rubric file from disk into a single string. |
| `build_prompt()` | `(template, data)` | Inject named list `data` into `{{placeholder}}` patterns. All placeholders must be present in `data`. Errors if any missing. |
| `.template_placeholders()` | `(template)` | (internal) Extract unique placeholder names from a template string. |

### `R/api.R`
| Function | Signature | Role |
|----------|-----------|------|
| `call_openai()` | `(prompt, params, api_key, base_url)` | POST to OpenAI `/chat/completions`. Forces `response_format: json_object`. Retries on 429/500/502/503 (up to 4 tries, exponential backoff via `httr2::req_retry`). Returns raw JSON string. |

### `R/parse.R`
| Function | Signature | Role |
|----------|-----------|------|
| `parse_response()` | `(raw)` | Parse raw JSON string ŌåÆ named R list. Rejects JSON arrays and non-JSON. Domain-specific field validation belongs in calling scripts, not here. |

### `R/inspect.R`
| Function | Signature | Role |
|----------|-----------|------|
| `preview_prompt()` | `(template, data)` | Print compiled prompt with char/word counts. `data` is a named list. No API call ŌĆö cheap sanity check before spending credits. |
| `print_result()` | `(raw, scores)` | Pretty-print raw JSON and parsed scores side by side. |

### `R/scoring.R`
| Function | Signature | Role |
|----------|-----------|------|
| `score_one()` | `(template_path, data, params, api_key)` | Single-row convenience wrapper. `data` is a named list matching template placeholders. Returns `list(raw, scores)`. |
| `score_many()` | `(df, template_path, params, n, output_dir, pii_check, ...)` | Batch score a data frame. Template placeholders determine which columns are required. Appends one row per API call. Skip logic, error CSV, progress bar, PII scan of all placeholder columns. |
| `status()` | `(df, template_path, params, output_dir)` | Print scored/failed/pending counts for a dataset. |

Internal helpers (all `@noRd`): `.output_path()`, `.error_path()`, `.to_row(id, data, ...)`, `.append_csv()`, `.check_df(df, required_cols)`

## Key design decisions

**Generic engine, domain logic in scripts.**
Nothing in `R/` assumes a rubric structure. `parse_response()` returns whatever
JSON keys the model sends. Score validation, aggregation, and interpretation
live in user scripts or downstream analysis ŌĆö not in the package.

**`response_format: json_object` always on.**
Forces valid JSON output from the model. The rubric template must instruct the
model to respond in JSON (OpenAI requirement). This eliminates parsing ambiguity.

**One output file per rubric version.**
`score_many()` derives the output path from the template filename:
`prompts/rubric_v1.txt` ŌåÆ `<output_dir>/rubric_v1.csv`. Rubric version is
implicit in the filename and explicit in the `prompt_version` column of every
row. Never mix rubric versions in one CSV.

**Failures go to a separate `_errors.csv`.**
Per-row `tryCatch` in `score_many()`. Successful rows go to the main CSV;
failed rows go to `<output_dir>/<stem>_errors.csv`. The main CSV is always
rectangular and analysis-ready. Failed rows are automatically retried on the
next run (they are absent from the main CSV, so skip logic treats them as pending).

**Skip logic matches on `id` + `model`.**
A row is skipped if `id` and `model` already appear in the output CSV. This
means: restart safely after interruption, and scoring with a different model
appends new rows (useful for model comparison studies).

**`output_dir` is explicit, not hardcoded.**
Default is `"data"` for backward compatibility with the dev scripts, but
package users should specify their own directory. The directory is created
automatically if it does not exist.

**Templates are the scientific instrument.**
Version by name suffix (`rubric_v1.txt`, `rubric_v2.txt`). Never edit in place
once data has been collected against a version.

## Development workflow

### Load the engine locally (without installing)
```r
for (f in list.files("R", pattern = "\\.R$", full.names = TRUE)) source(f)
if (file.exists(".env")) readRenviron(".env")
```

### Run the tests
```r
Rscript tests/testthat.R
```
Currently 47 tests, all passing. Tests use `withr::with_tempdir()` for
isolation and mock `call_openai` via inline `assign`/`on.exit` against
`globalenv()` (the only reliable pattern for non-installed packages ŌĆö
`local_mocked_bindings` requires pkgload).

### Regenerate documentation
```r
roxygen2::roxygenise()
```
All `man/*.Rd` files and `NAMESPACE` are auto-generated. Do not edit them by
hand.

### Demo scripts (developers only, not shipped in package)
```
scripts/01_score_one_text.R   Score a single text, print subscores
scripts/02_inspect.R          Preview prompt + score three example texts
scripts/03_score_batch.R      Full batch demo with skip logic and status()
```

## Output CSV schema

Each row of `<output_dir>/<output_name>.csv`:

| Column | Source |
|--------|--------|
| `id` | From input `df$id` |
| *(score columns)* | One per JSON key returned by the model |
| `raw` | The exact JSON string from the model |
| `prompt_version` | The `output_name` passed to `score_many()` |
| `model` | From `run_params()` |
| `temperature` | From `run_params()` |
| `scored_at` | ISO 8601 timestamp |

Placeholder columns are intentionally excluded ŌĆö input texts are not written
to the output CSV to prevent accidental leakage when sharing results.

Error CSV `<output_dir>/<output_name>_errors.csv`:

| Column | Source |
|--------|--------|
| `id` | From input `df$id` |
| `error` | `conditionMessage()` of the caught error |
| `model` | From `run_params()` |
| `scored_at` | ISO 8601 timestamp |
